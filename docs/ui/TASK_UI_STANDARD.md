# Task UI Standard

> 关联全局 UI 规范：`docs/ui/README.md`
> 关联全局窗口规范：`docs/ui/WINDOW_UI_STANDARD.md`

本规范约束 OpenCove 中 Task 卡片与 Task 编辑体验的交互与视觉，目标是：

1. 让用户在画布内快速改任务（最少点击）；
2. 让复杂配置仍有完整编辑入口（可发现、可回退）；
3. 保证组件行为稳定、可测试、可复用。

## 1. 交互原则

### 1.1 就地优先（Inline-first）
- 高频修改（标题、任务需求）必须支持在 Task 卡片直接编辑。
- 点击可编辑区域即进入编辑态，不应强制弹出全屏/模态窗口。
- Inline 编辑需支持：
  - 失焦保存；
  - `Esc` 取消；
  - 对多行文本支持 `Ctrl/Cmd + Enter` 保存。

### 1.2 全量编辑入口（Modal as advanced editor）
- 完整编辑弹窗保留，用于编辑低频字段（优先级、标签、自动生成策略等）。
- 弹窗必须通过显式入口触发（如铅笔图标），避免误触。
- 删除操作保持二次确认。

### 1.3 可发现性
- 可编辑文本（标题、需求）悬停需有弱反馈（背景或颜色变化）。
- 关键操作按钮采用图标+tooltip/aria-label 保证语义可理解。

## 2. Task 卡片行为规范

### 2.1 头部（Header）
- 标题支持 inline 编辑。
- 右上角保留三个图标按钮：
  - 模板：打开提示词模板菜单（选择/新增模板，并前置插入到任务要求）；
  - 铅笔：打开完整编辑弹窗；
  - `×`：删除任务（触发确认弹窗）。

### 2.2 内容区（Requirement）
- 点击需求文本进入多行 inline 编辑。
- 需求为空时禁止提交，恢复到原值。

### 2.3 底部（Footer）
- 状态下拉、Assign、Run Agent 保持操作分区，不与编辑态冲突。
- 编辑态下不应阻断节点拖拽/缩放的基本能力。

## 3. 弹窗视觉规范

- 基础实现优先复用 `cove-window-*` 通用窗口类。

### 3.1 信息层级
- 标题 > provider/meta > 字段组 > 错误信息 > 操作按钮。
- 使用一致的间距体系（建议 6/8/10/12/14/18）。

### 3.2 按钮语义分层
- `ghost`：取消/关闭；
- `secondary`：AI 生成等辅助动作；
- `primary`：Create/Save/Apply 主动作。

### 3.3 输入框可用性
- focus 态必须有清晰描边和轻量外发光；
- error 区块必须可读且与普通 hint 明显区分。

## 4. 数据行为约束

- inline 标题保存：
  - 更新节点标题；
  - 更新 `task.updatedAt`；
  - 将 `task.autoGeneratedTitle` 设为 `false`。
- inline 需求保存：
  - 更新 `task.requirement`；
  - 更新 `task.updatedAt`。
- 所有 quick save 均需去除首尾空白，空字符串不覆盖原值。

## 5. 测试与选择器规范

- Task 关键操作需稳定的 `data-testid`：
  - `task-node-open-editor`
  - `task-node-open-prompt-templates`
  - `task-node-close`
  - `task-node-inline-title-trigger`
  - `task-node-inline-title-input`
  - `task-node-inline-requirement-trigger`
  - `task-node-inline-requirement-input`
- E2E 用例必须覆盖：
  1. 卡片内需求直接编辑并持久化；
  2. 铅笔按钮打开弹窗编辑；
  3. 删除确认流程。
